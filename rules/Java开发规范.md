# Java 开发规范



## 1. 命名规范

### 1.1 通用命名

- **【强制】** 命名严禁以`_`、`$`开始或结束，杜绝拼音与英文混合，更不允许直接使用中文。
- **【强制】** 类名使用大驼峰（`UpperCamelCase`），方法名、参数、变量使用小驼峰（`lowerCamelCase`），常量全大写、下划线分隔。
- **【强制】** 抽象类以`Abstract`或`Base`开头，异常类以`Exception`结尾，测试类以`Test`结尾，枚举类名加`Enum`后缀。
- **【强制】** 包名全小写，点分隔仅一个自然语义单词，使用单数形式。
- **【强制】** POJO类布尔属性不加`is`前缀，数据库字段使用`is_xxx`并在`resultMap`中映射。
- **【强制】** 数组声明`[]`紧跟类型，如`String[] args`。
- **【推荐】** 类型名词放词尾（`startTime`），设计模式体现在命名中（`LoginProxy`），接口方法不加`public`修饰。

### 1.2 分层命名

- 数据对象：`xxxDO`；数据传输对象：`xxxDTO`；展示对象：`xxxVO`；禁止使用`xxxPOJO`。
- Service/DAO方法：获取单个`get`，多个`list`，统计`count`，新增`save/insert`，删除`remove/delete`，修改`update`。

## 2. 代码格式

- **【强制】** 缩进使用4空格，禁止Tab。单行最大120字符，超出后换行缩进4空格，运算符、点号随下文换行。
- **【强制】** 大括号遵循K&R风格：左括号不换行，右括号独占一行；`if/for/while`即使单条语句也必须使用大括号。
- **【强制】** 关键字与括号间加空格，左大括号前加空格；二元/三元运算符两侧加空格；逗号、分号后加空格；方法名与调用的`(`之间不加空格。
- **【强制】** 文件编码UTF-8，换行符使用Unix格式，文件尾保留一空行，行尾无空格。
- **【强制】** 禁止通配符导入，按静态导入、第三方、`java.*`、`javax.*`分组。
- **【强制】** switch必须有`default`分支，case要么终止，要么注释`// fall through`。
- **【推荐】** 单个方法总行数不超过80行，不同逻辑间用空行分隔。

## 3. 编程实践

### 3.1 OOP规约

- **【强制】** 静态成员通过类名访问；重写方法必须加`@Override`；Object的equals避免NPE，使用常量或确定有值对象调用或`Objects.equals`。
- **【强制】** 所有POJO属性使用包装类型，RPC返回值/参数使用包装类型，局部变量使用基本类型。POJO不得设默认值。
- **【强制】** 整型包装类值比较用`equals`；浮点数等值比较用`BigDecimal`或误差范围；BigDecimal用`compareTo`比较，构造时优先用String入参。
- **【强制】** 金额以最小货币单位整型存储；定义DO类属性类型与数据库字段匹配。
- **【强制】** POJO必须实现`toString`，禁止同时存在`isXxx()`和`getXxx()`。
- **【强制】** 构造方法禁止加入业务逻辑，有初始化请用`init`方法。序列化类新增属性勿改`serialVersionUID`（不兼容升级除外）。
- **【推荐】** 类内方法顺序：公有/保护 > 私有 > getter/setter；setter仅赋值无业务逻辑；循环中字符串拼接用`StringBuilder`；慎用clone；访问控制从严。

### 3.2 日期时间

- **【强制】** 日期格式化年份用小写`y`，区分`M`/`m`、`H`/`h`。获取当前毫秒用`System.currentTimeMillis()`。
- **【强制】** 禁止在程序中使用`java.sql.Date/Time/Timestamp`，推荐Java 8 时间API。
- **【强制】** 禁止硬编码一年365天，月份使用枚举值或Calendar常量。

### 3.3 集合处理

- **【强制】** 覆写`equals`必须覆写`hashCode`。判断集合空用`isEmpty()`。初始化指定初始容量。
- **【强制】** `Arrays.asList`返回的集合不可修改；`subList`结果不可强转ArrayList，注意父集合修改导致子列表异常。
- **【强制】** 集合转数组用`toArray(new T[0])`；`addAll`必须对入参做NPE判断。
- **【强制】** 不要在foreach循环里remove/add元素，使用迭代器。
- **【强制】** `Collectors.toMap`注意key重复和value为null的NPE。
- **【推荐】** 遍历Map用`entrySet()`或`Map.forEach`；利用Set唯一性去重；注意各Map对null的支持差异。

### 3.4 并发处理

- **【强制】** 线程资源必须通过`ThreadPoolExecutor`显式创建，禁止用`Executors`。线程须指定有意义名称。
- **【强制】** `SimpleDateFormat`线程不安全，需加锁或使用`ThreadLocal`/Java 8时间API。自定义`ThreadLocal`用后须在finally中`remove`。
- **【强制】** 加锁顺序保持一致，避免死锁。锁与解锁必须在try-finally中成对出现，`lock()`在try外。
- **【强制】** 并发修改同一记录使用乐观锁（version）或悲观锁，资金类信息推荐悲观锁。
- **【推荐】** 双重检查锁定实现延迟初始化需使用`volatile`。多线程随机数使用`ThreadLocalRandom`。避免`Random`共享实例。

### 3.5 控制语句

- **【强制】** if/else/for/while/do必须使用大括号。switch对String类型参数须判空。
- **【强制】** 三目运算符注意自动拆箱导致的NPE。高并发避免用等值判断作为中断条件。
- **【推荐】** 条件表达式结果赋值给有意义的布尔变量提升可读性。避免取反逻辑，少用if-else嵌套（不超过3层），可用卫语句、策略模式替代。
- **【推荐】** 循环体外处理对象创建、数据库连接等。公开接口必须做入参保护。

### 3.6 工具类规范

- **【强制】** 工具类必须定义为`final`类，并声明唯一的`private`构造器（内部抛出`AssertionError`），严禁实例化与被继承。
- **【强制】** 工具类只允许包含静态常量与静态方法，不得拥有非`static final`的成员变量，确保无状态与线程安全。
- **【强制】** 公共方法必须对入参做防御性校验：引用类型使用`Objects.requireNonNull`判空；非法参数主动抛出`IllegalArgumentException`。
- **【强制】** 公共方法的返回值严禁为`null`，空列表、空字符串等均应返回安全的空对象（如`Collections.emptyList()`）。
- **【推荐】** 工具类按功能域拆分为独立的`XXXUtils`类（如`StringUtils`、`FileUtils`），避免将所有方法堆砌到单一`Util`或`CommonUtils`中。
- **【推荐】** 工具类方法应设计为纯函数：不依赖外部可变状态，不修改传入的可变参数；若必须修改需在命名上明确体现（如`sort`）并进行防御性拷贝。
- **【强制】** 优先复用 JDK、Apache Commons、HuTool 等成熟库中已有的工具方法，避免重复造轮子。
- **【参考】** 工具类命名以 `Utils` 或 `Helper` 结尾，包名统一使用`util`；可通过静态导入提升调用可读性，严禁通过继承工具类来复用其静态方法。
- **【参考】**部分情况下允许使用接口+静态方法当作工具类

## 4. 注释规范

- **【强制】** 类、属性、方法使用Javadoc（`/** */`）。抽象方法、接口方法必须描述功能。类必须加`@author`和`@date`。
- **【强制】** 方法内单行注释用`//`，多行用`/* */`。枚举字段必须有注释。
- **【推荐】** 注释说明“为什么”，保持更新。谨慎注释掉代码，如保留需说明原因。
- **【参考】** 特殊标记：`TODO`(待处理)、`FIXME`(错误)，必须标注标记人和时间。

## 5. 前后端交互

- **【强制】** API使用HTTPS，路径全小写下划线分隔，名词复数，无后缀。响应体使用JSON。
- **【强制】** 列表空返回`[]`或`{}`。错误响应包含HTTP状态码、`errorCode`、`errorMessage`、`userTip`。
- **【强制】** JSON字段key为lowerCamelCase。超大整数用String返回。时间格式统一`yyyy-MM-dd HH:mm:ss`。
- **【推荐】** URL传参不超过2048字节，body内容控制长度。服务器内部重定向用forward。

## 6. 异常与日志

### 6.1 异常处理

- **【强制】** 可预检查的RuntimeException不应通过catch处理（如NPE）。异常不用于流程控制。
- **【强制】** 捕获异常必须处理或抛出，finally中关闭资源。事务回滚需手动处理。finally块禁止return。
- **【强制】** 捕获异常类型需完全匹配或为父类；调用RPC、反射等捕捉`Throwable`。
- **【推荐】** 返回值可为null时需注释说明，调用方防NPE。优先使用自定义业务异常，避免直接抛`RuntimeException`/`Exception`。

### 6.2 日志规范

- **【强制】** 使用SLF4J等日志门面框架。日志文件保存至少15天，命名`appName_logType_logName.log`。
- **【强制】** 日志输出使用占位符，避免字符串拼接。trace/debug/info级别需先判断日志开关。
- **【强制】** 生产禁`System.out/err`和`e.printStackTrace()`。异常日志包含案发现场信息和堆栈。禁止用JSON工具将对象转String。
- **【推荐】** 谨慎记录日志，生产禁debug，warn用于用户参数错误。用户敏感信息脱敏。

## 7. 工程结构

### 7.1 应用分层

- **【推荐】** 分层：Web层（控制器）-> Service层（业务逻辑）-> Manager层（通用业务，对第三方封装、Service下沉、多DAO组合）-> DAO层（数据访问）。上层可跨层依赖。
- **【参考】** 领域模型：DO（数据对象）、DTO（数据传输）、BO（业务对象）、VO（视图对象）、Query（查询对象，禁止用Map传参）。
- **【参考】** 分层异常：DAO层抛`DAOException`不打日志；Service层必须记录日志；Web层绝不抛异常，跳转友好页面；开放接口返回错误码。

### 7.2 二方库依赖

- **【强制】** GAV规范：`GroupId`最多4级；`ArtifactId`为产品线-模块名；版本号使用主版本.次版本.修订号，禁止SNAPSHOT，禁止同groupId、artifactId不同version。
- **【强制】** 接口返回值禁止使用枚举类型。依赖库群统一版本变量。
- **【推荐】** 使用`<dependencyManagement>`管理版本，二方库不新增配置项，保持精简稳定。

### 7.3 服务器

- **【强制】** 远程调用必须设超时。
- **【推荐】** 高并发调小`time_wait`；调大文件句柄数；JVM设置`-XX:+HeapDumpOnOutOfMemoryError`，Xms和Xmx设为一致。服务线程池隔离。

## 8. 设计规约

- **【强制】** 存储方案和数据结构需评审并沉淀文档。识别弱依赖并设计降级预案。
- **【推荐】** 设计时明确系统边界、模块关系、非功能性需求及演化原则。类遵循单一职责，优先聚合/组合而非继承，依赖倒置（面向接口），开闭原则，公共逻辑抽取复用。
- **【参考】** 设计文档用于明确需求与后期维护，避免为设计而设计。可扩展性关键在于隔离变化点。代码并非全部文档。
